# Sprint 10 补充：存档卡片布局与窗口标题修正

## 当前阶段

- 当前 Sprint：Sprint 10：存档管理体验优化
- 本次任务：修复继续游戏页存档文字过长，并增强启动页面识别
- 下一阶段建议：Sprint 10 第二小步：手动重命名存档

## 当前阶段目标

- 存档名称、分数、长度、时间不能全部挤在一行。
- 存档项文字过长时不能溢出按钮。
- 用户要能确认当前看的窗口是哪个场景，避免旧窗口造成误判。

## 本阶段完成功能

- 继续游戏页的存档项从单行按钮改为双行存档卡片。
- 第一行显示序号和存档名称。
- 第二行显示分数、蛇身长度、更新时间。
- 存档名称过长时自动截断并追加 `...`。
- 窗口标题会显示当前场景，例如：
  - `贪吃蛇 - menu`
  - `贪吃蛇 - gameplay`
  - `贪吃蛇 - continue`
  - `贪吃蛇 - settings`

## 本阶段页面与 UI 完成情况

继续游戏页 `ContinueScene`：
- 页面标题：`继续游戏`，使用 `TextBlock` 渲染。
- UI 组件：
  - `TextBlock`：标题、滚动提示、存档卡片文字、底部状态提示。
  - 自绘存档卡片：替代原本单行 `Button`。
- 存档卡片：
  - 第一行显示 `序号 + 存档名称`。
  - 第二行显示 `分数 + 长度 + 更新时间`。
  - 选中、悬停时有不同高亮。
  - 点击卡片读取存档。
- 返回主菜单按钮：
  - 保持可点击和可选中。
- 键盘操作：
  - `W/S` 或方向键切换。
  - `Enter/Space` 读取。
  - `Delete` 二次确认删除。
- 鼠标操作：
  - 点击卡片读取。
  - 悬停高亮。
  - 滚轮浏览。

应用窗口 `App`：
- 新增窗口标题场景标识。
- 新进程启动后标题应显示 `贪吃蛇 - menu`。
- 如果看到其他标题，说明当前看到的是旧窗口或已切换场景的窗口。

## 本阶段完成设计

- `ContinueScene` 不再用单行 `Button` 展示存档项，而是自绘两行卡片。
- 卡片区域仍通过 `pygame.Rect` 做命中检测。
- 返回主菜单仍保持按钮式绘制。
- `App` 新增 `_update_window_title(scene_name)`，每次场景切换后更新标题。

## 本阶段数据流设计

- 启动程序：`main.py` -> `App()` -> `self.scene = MenuScene` -> `_update_window_title("menu")`。
- 场景切换：`App.change_scene()` / `start_new_game()` / `load_gameplay_from_state()` -> 更新 `self.scene` -> `scene.on_enter()` -> `_update_window_title(scene_name)`。
- 渲染存档：`ContinueScene.render()` -> `SaveSlot` -> `_draw_save_card()` -> 两行展示存档摘要。
