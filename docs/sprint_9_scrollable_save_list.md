# Sprint 9 补充：继续游戏页可滚动多存档列表

## 当前阶段

- 当前 Sprint：Sprint 9：输入服务抽象与可配置键位
- 本次补充任务：增强继续游戏页多存档 UI，让大量存档可以浏览和读取
- 下一阶段建议：Sprint 9 第二小步：设置页展示当前键位配置

## 当前阶段目标

- 明确存档数据支持无限追加，不限制为 5 个。
- 继续游戏页虽然一次只展示 5 个可视项，但必须支持滚动浏览全部存档。
- 鼠标用户也要能查看第 6 个及以后的存档。

## 本阶段完成功能

- 继续游戏页支持鼠标滚轮滚动存档列表。
- 键盘 `W/S` 或方向键选择存档时，仍会自动滚动可见窗口。
- 页面新增滚动状态提示，例如 `显示 1-5 / 12  滚轮或 W/S 浏览更多存档`。
- 当前存档数据结构不变，仍支持任意数量存档。
- 每条存档仍使用 `Button` 组件渲染和点击。

## 本阶段页面与 UI 完成情况

继续游戏页 `ContinueScene`：
- 页面标题：`继续游戏`，使用 `TextBlock` 渲染。
- UI 组件：
  - `TextBlock`：页面标题、选择提示、滚动状态提示。
  - `Button`：每个可见存档项。
  - `Button`：返回主菜单。
- 存档按钮：
  - 最多同时显示 5 个，这是可视窗口大小，不是存档数量上限。
  - 点击某个存档按钮后读取对应存档。
  - 鼠标悬停某个存档按钮后同步高亮和选中。
- 键盘操作：
  - `W/S` 或上下方向键切换存档。
  - 选择移动到窗口边界时自动滚动。
  - `Enter/Space` 读取当前选中存档。
  - `Esc` 返回主菜单。
- 鼠标操作：
  - 滚轮向下：浏览更旧或更靠后的存档。
  - 滚轮向上：浏览更前面的存档。
  - 左键点击存档按钮：读取存档。
- 状态提示：
  - 有存档时显示当前可见范围和总存档数。
  - 无存档时显示无存档提示。

## 本阶段完成设计

- 存档数据层仍由 `SaveService.list_saves()` 返回完整存档列表。
- UI 层通过 `scroll_offset` 控制当前可视窗口。
- `MAX_VISIBLE_SAVES = 5` 表示一次最多绘制 5 个按钮，不表示存档数量限制。
- 鼠标滚轮通过 `pygame.MOUSEWHEEL` 调整 `scroll_offset`。
- 键盘选择通过 `_sync_scroll_to_selection()` 保证选中项总在可视窗口内。

## 本阶段数据流设计

- 继续游戏页进入：`ContinueScene.on_enter()` -> `SaveService.list_saves()` -> 得到完整存档列表。
- 鼠标滚轮：`pygame.MOUSEWHEEL` -> `_scroll()` -> 更新 `scroll_offset` -> 渲染新的 5 个可见存档按钮。
- 键盘选择：`InputService.just_pressed("menu_down")` -> `_move_selection()` -> `_sync_scroll_to_selection()` -> 自动滚动可见窗口。
- 点击读取：`Button.contains()` -> `_activate_selected()` -> `App.load_gameplay_from_state(game_state, save_id)`。
