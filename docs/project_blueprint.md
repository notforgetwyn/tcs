# Python + Pygame 贪吃蛇项目蓝图

## 1. 协作规则

本项目按以下规则推进：

1. 先输出简洁但专业的需求分析文档。
2. 再输出项目架构设计，包括项目目录结构、项目架构设计、页面设计、UI 设计、模块划分、类设计、数据流设计。
3. 然后把项目拆成多个迭代版本：MVP、菜单、设置、存档、优化。
4. 每次只完成一个小阶段，不一次性写完整个项目。
5. 每个阶段都先说明目标、实现思路、涉及文件，再给代码。
6. 所有代码必须可运行、结构清晰、便于扩展，禁止把所有逻辑写进一个文件。
7. 优先使用面向对象设计，加入必要注释，但不过度冗余。
8. 配置和存档统一使用 JSON 文件。
9. 写代码时考虑异常处理、边界情况和后续扩展。
10. 每完成一个阶段，都给出下一阶段建议。

额外约束：

- 先从最小可运行版本开始：蛇移动、食物、计分、游戏结束。
- 然后再逐步加入主菜单、设置、存档。
- 用户说“继续下一步”后，才进入下一个阶段。
- 用户说“重构”后，按工程化方式优化当前结构。
- 用户说“解释”后，用适合初学者的方式解释代码。

## 2. 需求分析

### 2.1 项目目标

开发一个基于 Python + Pygame 的桌面端贪吃蛇游戏，具备可玩性、可维护性和可扩展性，并按照工程化迭代方式逐步建设。

### 2.2 核心功能需求

1. 贪吃蛇支持上、下、左、右移动。
2. 蛇吃到黄色食物后增加分数。
3. 游戏过程中实时显示分数。
4. 游戏启动后先进入开始界面，而不是直接开始。
5. 开始界面支持：
   - 开始游戏
   - 继续游戏
   - 设置
   - 退出游戏
6. 支持游戏存档与读档。
7. 设置页面支持调整蛇移动速度，并持久化保存。
8. 整体架构支持后续扩展，例如暂停、皮肤、音效、障碍物、排行榜等功能。

### 2.3 非功能需求

1. 代码结构清晰，模块职责明确。
2. 采用面向对象设计。
3. 配置与存档使用 JSON 存储。
4. 具备基本异常处理能力，例如配置文件缺失、存档损坏、非法数据回退默认值。
5. 便于测试和后续重构。

### 2.4 边界与风险

1. 输入方向切换时，不能直接反向撞向自己。
2. 食物生成不能落在蛇身体上。
3. 配置文件或存档文件不存在时，需要自动创建或回退默认值。
4. 读档时需要校验数据合法性，避免程序崩溃。
5. 游戏速度调整需要设置合理上下限，避免不可玩。

### 3.2 架构设计思路

采用“应用层 + 场景层 + 领域模型层 + 基础设施层”的轻量分层：

1. `app.py`
   - 管理程序启动、Pygame 初始化、主循环切换。
2. `scenes/`
   - 管理不同页面，例如主菜单、游戏中、设置页、结束页。
3. `models/`
   - 存放核心游戏对象，例如蛇、食物、游戏数据。
4. `core/`
   - 提供通用能力，例如状态机、JSON 文件读写。
5. `ui/`
   - 提供按钮、文本等可复用界面组件。
6. `constants.py`
   - 统一管理窗口尺寸、颜色、网格大小、默认速度等常量。

该结构可以避免把逻辑塞进单文件，也方便后续加入暂停页、音效系统、排行榜系统等新模块。

### 3.3 页面设计

#### 启动流程

1. 启动程序
2. 进入主菜单
3. 从主菜单进入游戏 / 继续游戏 / 设置 / 退出

#### 页面列表

1. `MenuScene`
   - 显示游戏标题
   - 显示开始游戏、继续游戏、设置、退出游戏按钮
2. `GameplayScene`
   - 显示地图、蛇、食物、当前分数
   - 处理移动、碰撞、得分、结束判定
3. `SettingsScene`
   - 调整蛇速度
   - 保存设置到 JSON
4. `GameOverScene`
   - 显示最终得分
   - 支持返回菜单或重新开始

### 3.4 UI 设计

初版采用简洁风格，强调结构清晰和后续可扩展：

1. 主色调
   - 背景：深灰/黑色
   - 蛇：绿色
   - 食物：黄色
   - 文本：白色
   - 按钮高亮：蓝色或浅绿色
2. 布局原则
   - 页面元素居中对齐
   - HUD 分数固定显示在顶部
   - 菜单按钮垂直排列
3. 扩展预留
   - 后续可替换字体、增加图标、动效和音效

### 3.5 模块划分

#### 应用控制模块

- `main.py`: 程序入口
- `app.py`: 应用主循环与场景切换
- `game.py`: 组织游戏整体依赖

#### 业务模型模块

- `snake.py`: 蛇的位置、移动、增长、碰撞逻辑
- `food.py`: 食物生成与重置
- `game_data.py`: 分数、速度、运行状态、存档结构

#### 页面模块

- `menu_scene.py`: 主菜单行为
- `gameplay_scene.py`: 核心玩法
- `settings_scene.py`: 设置页
- `game_over_scene.py`: 结束页

#### 基础设施模块

- `state_machine.py`: 场景切换
- `file_manager.py`: JSON 配置和存档读写

#### UI 模块

- `button.py`: 按钮组件
- `text.py`: 文本渲染封装

### 3.6 类设计

#### `App`

职责：
- 初始化 Pygame
- 创建窗口和时钟
- 驱动主循环
- 分发事件到当前场景

关键方法：
- `run()`
- `change_scene(scene_name)`

#### `BaseScene`

职责：
- 统一场景接口

关键方法：
- `handle_event(event)`
- `update(delta_time)`
- `render(screen)`
- `enter()`
- `exit()`

#### `GameplayScene`

职责：
- 管理一局游戏
- 协调蛇、食物、分数与碰撞

关键属性：
- `snake`
- `food`
- `score`
- `is_game_over`

#### `Snake`

职责：
- 管理身体坐标
- 控制移动方向
- 执行移动和增长
- 判断自撞

关键方法：
- `set_direction(direction)`
- `move()`
- `grow()`
- `check_self_collision()`

#### `Food`

职责：
- 在合法网格内生成位置

关键方法：
- `respawn(occupied_positions)`

#### `FileManager`

职责：
- 读写 `settings.json` 和 `save.json`
- 做默认值回退和异常兜底

关键方法：
- `load_settings()`
- `save_settings(data)`
- `load_save()`
- `save_game(data)`

### 3.7 数据流设计

#### 运行时数据流

1. 用户按键输入
2. 当前场景接收事件
3. `GameplayScene` 修改蛇方向
4. 游戏循环触发 `update`
5. 蛇移动
6. 判定是否吃到食物
7. 若吃到：
   - 分数增加
   - 蛇增长
   - 食物重新生成
8. 判定是否撞墙或撞自己
9. 若结束：
   - 写入本局结果
   - 进入结束页或菜单页

#### 配置数据流

1. 程序启动时读取 `config/settings.json`
2. 若文件不存在或非法，则加载默认配置
3. 设置页修改速度
4. 保存配置到 JSON
5. 下次启动继续使用该配置

#### 存档数据流

1. 玩家手动开始新游戏或自动进入游戏
2. 游戏中定时或在关键时机写入 `data/save.json`
3. 主菜单点击继续游戏
4. 从 `save.json` 恢复蛇位置、方向、分数、速度等状态

## 5. 第一阶段开发计划

### 5.1 阶段目标

完成一个最小可运行的单局游戏版本，具体包括：

1. 玩家可以使用方向键控制蛇移动
2. 蛇可以吃到黄色食物
3. 分数实时增长并显示
4. 撞墙或撞自己后游戏结束

### 5.2 实现思路

第一阶段不引入完整场景系统，但代码结构仍按后续可扩展方式组织：

1. 先建立基础项目目录
2. 搭建 `main.py + app.py + models + constants`
3. 先只实现 `GameplayScene` 所需最小集合
4. 使用网格坐标而不是像素坐标管理蛇和食物，简化逻辑
5. 使用 Pygame `Clock` 控制刷新率，使用“定时步进”控制蛇速度
